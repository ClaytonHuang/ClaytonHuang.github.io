  • 查看服务器的系统版本

    cat /proc/version

  • 列出linux所有版本信息(Linux Standard Base)

    lsb_release -a

  • 查看cpu信息

    cat /proc/cpuinfo |more

  • 查看系统内存信息

    cat /proc/cupinfo |more

  • 查看服务器的磁盘信息

    fdisk -l
cat /proc/partitions

PS: /proc/是一个伪文件系统，存在于内存中，不占用外存空间，可以在运行时访问内核内部数据结构、改变内核设置。

ssh登录配置

创建系统账号并修改密码

腾讯云默认登录账户ubuntu，密码在控制台可修改，同时可以创建其他账户

ssh 登录

ssh ubuntu@your_server_ip

不想每次ssh都输入密码？

其中，ssh user@host表示登录远程服务器；

单引号中的mkdir -p .ssh && cat >> .ssh/authorized_keys表示登录的远程服务器用户主目录下，如果不存在就创建一个.ssh文件夹；然后将 < 后的内容追加到authorized_keys文件下

~/.ssh/id_rsa.pub就是本地机器的公钥

这样便将本地主机的公钥存入了远程服务器，之后便不用密码登录，远程服务器会用随机密码和本地交互，以判断本地公钥是否正确，即是否可以登录。

域名解析

在header中选择域名管理，域名实名验证（一般3-5天）后，即可进行DNS解析，登录更方便，不用每次输入ip地址啦。

部署ssl文件到服务器

腾讯云官方指引

采用Apache 2.x部署证书

不要问为什么，nginx用多了，换个口味。



操作步骤

  1. 安装apache2

    sudo apt-get install apache2

    启动服务

    sudo  /etc/init.d/apache2 [ start | stop | restart | status ]

  2. 安装Openssl模块

    opensll

  3. 加载Apache的SSL模块

    sudo a2enmod ssl

    随后会在/etc/apache2/sites-available下生成default-ssl.conf文件

    • 修改VirtualHost 的domain标签
    • 添加ServerName www.domain.cn
    • 修改 SSLCertificateFile/KeyFile/ChainFile

4. 将default-ssl.conf软链接到/etc/apache2/sites-enabled

sudo ln -s /etc/apache2/sites-available/default-ssl.conf /etc/apache2/sites-enabled/001-ssl.conf

重新加载Apache配置文件

sudo /etc/init.d/apache2 force-reload

重启Apache服务

sudo /etc/init.d/apache2 restart

Q&A

ssh root Permission denied?

在ssh root@host时，密码输入正确，但是仍然无法登录远程服务器，提示Permission denied?

A: 修改 /etc/ssh/sshd_config并重启服务

ssl证书只要配置在443端口?

ssl 默认端口443，而在其他端口要带端口号，不方便。
