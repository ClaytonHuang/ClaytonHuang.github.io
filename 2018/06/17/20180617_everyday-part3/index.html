微信小程序「日更小图文」(下)

发表于 2018-06-17 | 分类于 product

前言

按原计划，日更小图文三步走，后端、前端和拓展。

需求拓展

  1. 【图翁的每日推荐已经不再更新】需要先遍历资源中的图文链接，确定好每天爬取的顺序；

  2. 用docker做服务端的部署；

  3. 在微信小程序中添加tabBar，提供历史文章的浏览。

具体实现

获取每日链接

  1. 爬取life, illustrations, culture, picture分类下的所有图文。

  2. 通过async-request和cherrio组合，将爬取的链接存入列表。

  3. 每天取用一个，做基于Redis list做环形数据结构。

    可以使用RPOPLPUSH key key对同一个列表做回环

docker的简单应用

使用docker

docker是目前最主流的容器工具，其口号就是：build once, deploy anywhere。将部署环境container化的设计，可以减少移植、发布的工作量。

具体步骤

  • 将server打包为images

    参考

    $ cd node-docker
$ docker build -t node-docker-prod .
$ docker run --rm -it -p 8080:8080 node-docker-prod

  • 引入redis，在server和redis间建立network

    server无法找到127.0.0.1:6379下的redis?
    参考

    docker network create redis
docker run -d --net "redis" --name redis redis
docker run -d -p 8100:8100 --net "redis" --name node redis-node

    PS: 在redis连接时，要用 redis-container-name 代替localhost

  • 引入docker-compose，将redis和everyday-server放到默认的network下

  • 将生成的images push到docker hub

    之前要通过docker tag修改images name，符合 username/images规范才能push

微信小程序升级

  • 通过wxParse解析HTML及markdown?

    wxParse，支持将HTML&markdown解析。

  • 用Promise封装wx.request方法

    const $get = (url, data) => {
return new Promise((resolve, reject) => {
    wx.request({
    url,
    data,
    header: { 'Content-Type': 'json' },
    success: resolve,
    fail: reject
    })
  })
}

  • mpvue小程序内的页面跳转传参

    通过wxmini的自带方法：

    在非tabBar页面跳转，用wx.navigateTo || wx.redirectTo (url)
在tabBar页面跳转，用wx.switchTab(url)

    通过构造带query的url，再在目标pages的onLoad中解析

  • vue的sub component $emit和父组件的v-on:your_emitname交互

    实现中遇到的一个小问题，是如何实现在父组件中触发子组件的onClick事件
    后来参考官方文档的emit事件回调解决了

    • 方案

      子组件中，绑定@click=”eventFunction”
      在methods中添加eventFunction，同时通过this.$emit(‘eventName’, {args})传递事件名和参数。

      父组件中，调用v-on:eventName=”eventFunction”
      然后在methods中实现eventFunction即可触发子组件中的响应。

      通过这个方案，也可以在父子组件中实现数据交互。
