微信小程序「日更小图文」(上)

发表于 2018-06-07 | 分类于 Product ， nodejs

前言

想起来，已经很久没有做过系统性开发的训练，作为一名伪全栈和科研民工，本人决定忙里偷个闲。

于是打算做一个每日更新文章的微信小程序，希望以后突然想起时，打开就可以看到不一样的风景（￣。。￣）

完美。

然而事实是完成后可能又是一个有各种问题的辣鸡，(:з」∠)，但是好歹要有些皮格马利翁效应吧？？

需求分析

项目：everyday

可以每日分享一张美图美文的微信小程序。

后端：everyday-server

爬取某个每日定时更新的图文网站，或有较丰富图文资源的网站，获取内容（图片或html数据），定为图翁。

前端：everyday-wxapp

微信小程序，显示获取的图文数据。
附加能力：分享、登录。

后端设计

nodejs + redis

NPM组件

koa, koa-router, koa-static, cheerio, phantomjs, node-schedule, moment

工作重点

  1. 通过koa建立api和静态服务框架

  2. 爬取每日图文并转html为图片

    通过http获取页面html，利用cheerio类jquery操作，再通过phantom转html网页为.jpg图片

  3. 将图片本地化，同时将链接保存到redis

    fs.writeSync到static目录下，redis还没有用上；api中获取今日图片的接口是抽象的。

  4. 每天定时执行

    node-schedule + cron表达式

  5. 提供接口，前端->redis->今日图文链接

ps: 其实可以不用做得这么麻烦，但是为了学习–+

Q&A

  • http获取的网页数据，编码GBK，汉字保存到本地后变为十六进制显示？

    A： 通过icon-lite进行编码转换

  • error while loading shared libraries: libfontconfig.so.1: cannot open shared object file: No such file or directory

    A: sudo apt-get install libfontconfig

  • 访问koa static 正常，但访问api，浏览器显示 Malicious Path？

    没有对end /进行映射，参考

  • phantomjs 抓取和截取中文网页乱码？

    sudo apt-get install xfonts-wqy
