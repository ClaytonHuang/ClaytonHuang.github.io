Vue.js基础

Vue的优势：不用管DOM操作，所有的代码只需要关注逻辑层；组件化应用构建。

基本内容

  • 指令： v-xxxx，以表示它们是Vue提供的特殊特性
    v-on:click 变量层: methods

  • 条件与循环：

    v-if=’variable’
    v-for=’var in vars’

  • 表单与应用状态之间的双向绑定

    v-model=’message’

  • 组件构建

    Vue.component('todo-item', {
props: ['todo'],
    template: '<li>{{ todo.text }}</li>'
})

风格指南

基本原则：
JS中常用camelCase
HTML中常用kebab-case

  1. Vue组件名必须为多个单词
  2. 组件的data必须为一个函数

  3. props定义尽可能详细

    props: {
    status: String
}

  4. 为v-for设置键值key

  5. 为组件样式设置作用域

还有很多，参考文档

Redis

简介

Redis 是一个开源的key-value存储系统，通常被称为数据结构服务器，因为存于其中的值可以是String, Map, list, sets和sorted sets等等。

Ubuntu下配置Redis

推荐直接apt-get安装redis-server，参考

应用

目前暂时会用到的两个数据结构，就是Key-String和Key-Hashmap的读写

通过node连接操作redis，有很多库，如node-redis, ioredis，考虑到异(neng)步(li)问题，项目里用了async-redis.

  • Key-String

    通过set, get操作即可

    set key value
get key

  • Key-Hashmap

    redis hash 其实是string类型的field和value的映射表，这里是用来存放历史图文和对应的图像尺寸等信息，由于后期文章数量可能会比较多，因此想用hashmap提高效率。

    后来才发现多是不可能多的，这几个月是不可能多的。反正又上不了线，就是只能本地调试，才能跑一跑这样子。

    存放数据

    hmset key field1 value1 field2 value2

    还有其他几种写法，但关键，是将field和value对应起来，然后绑定到key下

    显示hashmap中所有数据

    hgetall key

    删除某个字段

    hdel key

    …
